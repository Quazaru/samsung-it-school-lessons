 Конспект:
 	Анонимный класс можно создавать прямо в методах класса или в самом теле класса, единственное его ограничение - невозможность создания static методов.
 	
 	Мы можем создать интерфейс в нашем проекте и потом создать его экземпляр там, где нам нужно, так же в методе или в самом теле класса. ГЛавное преимущество - мы можем не создавать отдельный класс для работы с определёнными методами. 
 	Мультипоточность может помочь в одновременнном отображении гуи и действия программы
   
   Потоки:
   	class x extends Throw {}
   		@Override
   		public void run() {
   		   // что будет делать класс после создания с помощью start();
   		}
   		
   	Throw.sleep(ms); // throws InterruptedException error
